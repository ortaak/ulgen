### Mimari Kararlar

**Circular dependency detection — BFS**
Server-side, POST isteği sırasında çalışır. Yeni `blockingCardId → blockedCardId` bağlantısı
eklendiğinde, `blockedCard`'ın mevcut "blockingFor" zincirini BFS ile gezer.
Zincirde `blockingCardId`'yi bulursa döngü var demektir → 400 döner.

BFS seçilmesinin nedeni: DFS'e kıyasla bellek açısından öngörülebilir, büyük grafiklerde
cycle detection için industry standardı.

**`_count` ile kart rozeti**
Kart listesi (board yüklenirken) `_count.blockedBy` ve `_count.blockingFor` alanlarını
Prisma üzerinden çeker. Tek bir sorgu, N+1 problemi olmaz.
Timeline için aynı `_count` pattern'ı `TASK_INCLUDE` ile uygulandı.

**DependencyPicker — board store'dan kart listesi**
Picker, `useBoardStore`'dan `board.lists` üzerinden tüm kartlara erişir.
Ayrı bir endpoint çağrısı yapmaz. Mevcut board kartları zaten store'da olduğundan sıfır ek request.

### Karşılaşılan Sorunlar & Çözümler

## 🧪 Manuel Test Senaryoları

1. **Bağımlılık ekleme**
   - Kart detay modalı aç → "Bağımlılıklar" → "Ekle"
   - Bir kart seç → beklenen: "Bitmesini beklediği kartlar" listesinde görünür
   - Kart önizlemesinde kırmızı kilit rozeti görünür

2. **Bloklanan kartın rozeti**
   - A kartına B'yi bağımlı yap (B, A'yı bekliyor)
   - B kartının önizlemesinde kırmızı kilit (blokedByCount=1) görünür
   - A kartının önizlemesinde sarı kilit (blockingForCount=1) görünür

3. **Dairesel bağımlılık engeli**
   - A → B bağımlılığı ekle (B, A'yı bekliyor)
   - B → A bağımlılığı eklemeyi dene → "Dairesel bağımlılık" hatası alınmalı

4. **Bağımlılık silme**
   - Modal'da X butonuna bas → bağımlılık listeden kalkar
   - Kart rozeti kaybolur (board yenilenince)

5. **Timeline bloke uyarısı**
   - Bloke edilmiş kart timeline'a ekle → kırmızı uyarı bandı görünür

6. **Tamamlanan kart**
   - Bloklayan kart (A) tamamlandı olarak işaretle
   - B kartının modali → A'nın yanında ✓ görünmeli, üzeri çizili

## 📚 Öğretici Notlar

### 💡 NEDEN BFS (Breadth-First Search)?

**Kullandığım pattern/teknik:** Graph cycle detection via BFS
**Ne işe yarar:** Directed graph'ta iki node arasında path var mı kontrol eder.
**Alternatif ne olabilirdi:** DFS (Depth-First Search) — daha az bellek kullanır ama iteratif
implementasyonu karmaşık. Union-Find (Disjoint Set Union) — statik graph'ta çok hızlı ama
dinamik eklemede karmaşık.
**Bu projeye neden daha uygun:** BFS, async/await ile yazması kolay (Prisma'dan her seferinde
bir seviye çekiyoruz). Card dependency graph'ları tipik olarak sığ (3-5 seviye) — BFS'nin
bellek dezavantajı pratikte önemsiz.
**Daha fazla öğrenmek için:** "Topological Sort", "DAG (Directed Acyclic Graph)", Kahn's Algorithm
